# Day 2 — From API Consumer to API Provider

## Goal

Build real HTTP APIs with Flask — moving from consuming external APIs (Day 1) to serving them.

> "Yesterday we used APIs — today we ARE the API"

## By End of Day 2, Students Will:

- Understand the full client ↔ server request/response lifecycle
- Build a Flask app with multiple routes returning JSON
- Handle query params and request body in Flask
- Apply Day 1 decorators (`@log_execution`, `@handle_errors`) inside a Flask app
- Return a uniform response structure from every endpoint
- Build a full CRUD in-memory REST API
- Separate HTTP routing from business logic (services layer)
